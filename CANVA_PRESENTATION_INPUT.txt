FLUJO DE AUTENTICACIГ“N:
1. Usuario ingresa email/password
2. Backend valida y genera JWT
3. Frontend almacena token en localStorage
4. Cada request incluye token en header Authorization
5. Backend verifica JWT y autoriza acciГіn
6. Respuesta JSON con datos o error

FLUJO DE SUBIDA DE DUMP:
1. Usuario selecciona archivo
2. Frontend valida (tamaГұo, extensiГіn)
3. POST /dumps con form-data
4. Backend guarda archivo en uploads/
5. Registra en BD (DroneDump table)
6. Frontend obtiene parsed dump JSON
7. Muestra anГЎlisis en interface

7. API REST - ENDPOINTS PRINCIPALES
================================================================================

AUTENTICACIГ“N (/auth)
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
POST /auth/register
  Request: { email, password }
  Response: { email, access_token, token_type }
  Status: 201 (Created)

POST /auth/login
  Request: { email, password }
  Response: { access_token, token_type, expires_in }
  Status: 200 (OK)

GET /auth/me
  Headers: Authorization: Bearer <token>
  Response: { email, created_at }
  Status: 200 (OK)

DRONES (/drones)
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
GET /drones
  Headers: Authorization: Bearer <token>
  Response: [ { id, brand, model, drone_type, owner_email } ]
  Status: 200

GET /drones/{id}
  Headers: Authorization: Bearer <token>
  Response: { id, brand, model, drone_type, notes, owner_email }
  Status: 200

POST /drones
  Headers: Authorization: Bearer <token>
  Request: { brand, model, drone_type, notes }
  Response: { id, brand, model, owner_email }
  Status: 201

PUT /drones/{id}
  Headers: Authorization: Bearer <token>
  Request: { brand, model, drone_type, notes }
  Response: { id, ... }
  Status: 200

DELETE /drones/{id}
  Headers: Authorization: Bearer <token>
  Status: 204 (No Content)

DUMPS (/dumps)
в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
GET /drones/{drone_id}/dumps
  Headers: Authorization: Bearer <token>
  Response: [ { id, filename, size, created_at } ]
  Status: 200

POST /dumps
  Headers: Authorization: Bearer <token>
  Request: FormData { drone_id, file }
  Response: { id, filename, size }
  Status: 201

GET /drones/{drone_id}/dumps/{dump_id}/parse
  Headers: Authorization: Bearer <token>
  Response: { parsed_data: {...} }
  Status: 200

DELETE /drones/{drone_id}/dumps/{dump_id}
  Headers: Authorization: Bearer <token>
  Status: 204

в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

1. Clonar repositorio
   $ git clone https://github.com/AgustinHi/tfm-drones.git
   $ cd tfm-drones

2. Crear base de datos MySQL
   $ mysql -u root -p
   > CREATE DATABASE tfm_drones CHARACTER SET utf8mb4;

3. Backend
   $ cd backend
   $ python -m venv .venv
   $ .\.venv\Scripts\activate          (Windows)
   $ source .venv/bin/activate          (macOS/Linux)
   $ pip install -r requirements.txt
   $ echo "DATABASE_URL=mysql+pymysql://root:@localhost/tfm_drones" > .env
   $ echo "JWT_SECRET=tu_clave_super_secreta_aqui_minimo_32_chars" >> .env

4. Frontend
   $ cd frontend
   $ npm install
